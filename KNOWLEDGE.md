# KNOWLEDGE.md

Единый источник настроек пайплайна отслеживания цен портфеля Финам. **Все
skill'ы (`extract_price`, `tracker`, `telegram_alert`) берут значения по умолчанию
отсюда.** Явно переданные skill'у аргументы имеют приоритет над значениями из
этого файла. Чтобы изменить поведение пайплайна — правь этот файл, а не skill'ы.

## Правило алерта

- `threshold` (порог срабатывания) — **`0.10`** (10%).
- `direction` (направление) — **`both`** (алерт и при росте, и при падении).
- Условие срабатывания по бумаге:
  `abs(current − averagePrice) / averagePrice ≥ threshold`,
  где `current` — текущая цена (из `extract_price`), `averagePrice` — средняя
  цена покупки (из Finam MCP `get-account`).
- Направление элемента: `down` при `current < averagePrice`, `up` при
  `current > averagePrice`. При `direction=down` учитываются только падения, при
  `direction=up` — только рост, при `both` — любое отклонение ≥ порога.

## Хранилище snapshot (репозиторий данных)

- `repo` — **`mikaelo/tracker-data`**.
- `branch` — **`main`**. Snapshot'ы пишутся **напрямую в `main`**, без создания
  веток и без PR.
- Путь файла — **`<account_id>/YYYY-MM-DD-HH-MM.json`** (папка на счёт), дата/время
  берутся из `generated_at` в **UTC** (напр. `869168/2026-07-14-14-15.json`).
  Раскладка по папкам на счёт исключает коллизии имён и разделяет базу сравнения
  для разных счетов.
- Запись — GitHub MCP `create_or_update_file`. Имя уникально в пределах минуты →
  это всегда новый файл, `sha` не нужен.

## Правило записи и возврата snapshot (tracker)

Разделяем **что сохраняется** и **что возвращается**:

- **Сохранение — всегда и полностью.** Snapshot содержит **ВСЕ** бумаги счёта
  (open, current, datetime) и **всегда** записывается в репозиторий-хранилище
  целиком, со всеми бумагами. Запись ничем не фильтруется и не гейтится — полный
  snapshot нужен, чтобы у следующего запуска была полная база для сравнения по
  цене открытия.
- **Возврат skill — только бумаги, попадающие под условия.** В ответе (JSON +
  Markdown-таблица) `tracker` возвращает **ТОЛЬКО** те бумаги, для которых
  выполнены **ОБА** условия:
  1. **Отклонение от цены покупки** (по текущей цене):
     `abs(averagePrice − current) / averagePrice > snapshot_deviation_threshold`
     (`snapshot_deviation_threshold` = **`0.10`**, т.е. 10%);
  2. **Изменение цены относительно предыдущего сохранённого snapshot** больше
     `snapshot_change_threshold` (= **`0.01`**, т.е. 1%). **Базис сравнения
     зависит от дня** предыдущего snapshot относительно текущего:
     - **разные дни** (предыдущий snapshot — за другой торговый день) →
       сравнивай по **цене открытия**:
       `abs(open − prev_open) / prev_open > snapshot_change_threshold`;
     - **один день** (предыдущий snapshot — за тот же день) → сравнивай по
       **текущей цене**:
       `abs(current − prev_current) / prev_current > snapshot_change_threshold`.
     Здесь `open`/`current` — из текущего snapshot, `prev_open`/`prev_current` —
     цена открытия / текущая цена той же бумаги в предыдущем сохранённом snapshot.
- **Определение дня.** День snapshot — календарная дата **по московскому времени
  (МСК, UTC+3)**, полученная из `generated_at` (UTC + 3 часа). Сравнивай дату
  текущего snapshot с датой предыдущего.
- **Предыдущий snapshot** — самый свежий файл `YYYY-MM-DD-HH-MM.json` **в папке
  `<account_id>/`** того же счёта (имена сортируются хронологически), взятый
  **ДО** записи текущего. Снапшоты других счетов в сравнение не берутся.
- **Первый запуск / нет предыдущего значения** (репозиторий пуст или бумага
  новая): условие 2 для такой бумаги считается выполненным (фильтр по возврату
  идёт только по условию 1).

## Котировки и формат

- Источник котировок — Finam MCP `get-quote`.
- Биржа по умолчанию — **`@MISX`** (Московская биржа). Если в тикере нет `@MIC`,
  добавляется `@MISX`: `SBER` → `SBER@MISX`.
- Время в колонках/полях `datetime` — **московское (МСК, UTC+3)** в формате
  ISO 8601 со смещением `+03:00` (время котировки `as_of` из ответа `get-quote`
  приходит в UTC и переводится в МСК, +3 часа).
- `generated_at` в snapshot — в **UTC** (из него строится имя файла).

## Классы бумаг

В пайплайне учитываются **только акции и облигации**; остальные классы (фонды/
БПИФ, фьючерсы, валюта и т.п.) исключаются из snapshot и алертов.

- Класс инструмента берётся из Finam MCP `get-instrument`, поле `category`:
  - `category` = «Акции» → **`stock`** (включая депозитарные расписки, напр. OZON);
  - `category` = «Долговые инструменты» → **`bond`**;
  - любое другое (`Инвестиционные фонды`, фьючерсы, валюта, ошибка) → **исключить**.
- По тикеру/названию класс **не определяем** — только по данным API (напр. `BOND`,
  `SBGB` — это фонды, а не облигации).
- **Иконки в Telegram-сообщении** по (класс, направление):
  - `stock`: рост — 🟢, падение — 🔴;
  - `bond`: рост — 🔵, падение — 🟠.

## Telegram

- `bot_token` — из переменной окружения **`TELEGRAM_BOT_TOKEN`**.
- `chat_id` — из переменной окружения **`TELEGRAM_CHAT_ID`**.
- Отправка — Telegram Bot API `sendMessage`. Если хотя бы один доступ не задан,
  сообщение не отправляется, а результат помечается `telegram.sent=false`.
- Секреты (сам токен) в этот файл и в вывод skill'ов не попадают — здесь только
  **имена** переменных окружения.
- Фактические значения хранятся в локальном файле `.env` в корне репозитория
  (`TELEGRAM_BOT_TOKEN` — токен бота, `TELEGRAM_CHAT_ID` — ID чата). `.env` в
  `.gitignore` и **не коммитится**; шаблон без секретов — `.env.example`.
- Отправка выполняется скриптом `send.py` (Telegram Bot API `sendMessage`).
- Разделение ответственности: `tracker` считает дифф и цифры; текст сообщения
  генерит агент `telegram-message` (дешёвая модель Haiku, только оформление);
  skill `telegram_alert` только отправляет готовый текст через `send.py`.
- **Сетевой доступ в облаке.** В облачной сессии Claude Code исходящий трафик
  идёт через прокси окружения, и по умолчанию (уровень **Trusted**)
  `api.telegram.org` **заблокирован** (403 на CONNECT). Чтобы отправка работала
  из облака, в настройках окружения выбери **Network access → Custom**, добавь
  в **Allowed domains** хост `api.telegram.org` и отметь галочку сохранения
  дефолтных хостов; затем открой **новую** сессию. Локально и в CI (где Telegram
  доступен) доступ включать не нужно.
