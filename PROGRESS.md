# PROGRESS

Пайплайн отслеживания цен портфеля Финам, собранный из компонуемых Claude Code
skill'ов (директория `.claude/skills/`). Каждый шаг — отдельный skill; шаги
переиспользуют друг друга, а не дублируют логику.

## Схема пайплайна

```
extract_price ──► tracker ──► telegram-message ──► telegram_alert
 (1 тикер)     (снапшот+дифф,    (агент Haiku:        (только
                расчёт цифр)      текст из диффа)      отправка)
```

## Состояние шагов

| # | Skill           | Что делает                                                                                              | Источник данных                          | Статус |
|---|-----------------|---------------------------------------------------------------------------------------------------------|------------------------------------------|:------:|
| 1 | `extract_price` | Котировка одного тикера `TICKER@MIC` → `open`, `current` (last), `datetime`.                             | Finam MCP `get-quote`                    | ✅ |
| 2 | `tracker`       | По счёту резолвит позиции, по каждой вызывает `extract_price`. **Полный** snapshot (все бумаги) всегда пишется **напрямую в `main`** `mikaelo/tracker-data` (`YYYY-MM-DD-HH-MM.json`, без веток/PR), а в ответе возвращаются **только** бумаги под условиями из `KNOWLEDGE.md` (откл. от цены покупки > 10% И изменение vs предыдущий snapshot > 1%: другой день → по цене открытия, тот же день → по текущей цене). В конце из диффа генерит текст (агент `telegram-message`, Haiku) и шлёт через `telegram_alert` — один вызов прогоняет весь pipeline. | Finam MCP `get-accounts-list`, `get-account` + skill `extract_price`; агент `telegram-message`; skill `telegram_alert`; GitHub MCP `get_file_contents`/`create_or_update_file` | ✅ |
| 3a | `telegram-message` (агент, Haiku) | Из готового диффа `[{ticker, average_price, current, change_pct, direction}]` формирует **только текст** Telegram-сообщения. Не считает, не ходит в сеть, не отправляет. | Дешёвая модель (Haiku), вход — дифф от `tracker` | ✅ |
| 3b | `telegram_alert` (skill) | **Только отправка**: принимает готовый текст и шлёт через `send.py`. Никакой логики/расчётов. | `send.py` (Telegram Bot API `sendMessage`) | ✅ |

## Ключевые решения

- **Единый источник настроек — `KNOWLEDGE.md`.** Все настройки пайплайна (порог
  10%, `direction`, счёт по умолчанию, репозиторий-хранилище и ветка, биржа по
  умолчанию, таймзона МСК, env-имена Telegram) вынесены в `KNOWLEDGE.md` в корне
  репозитория. Skill'ы берут значения по умолчанию оттуда; явные аргументы их
  перекрывают. Меняешь поведение — правишь `KNOWLEDGE.md`, а не skill'ы.
- **Переиспользование skill'ов.** `tracker` и `telegram_alert` берут текущую цену через
  `extract_price`, а не через `get-quote` напрямую. Цена покупки (`averagePrice`)
  доступна только из `get-account`.
- **Форма ответа `get-account` (Finam MCP).** Позиции — вложенные объекты:
  `positions[].symbol`, `positions[].averagePrice.value` (строка),
  `positions[].currentPrice.value` (строка). Значения-строки приводятся к числу.
- **`tracker`: сохраняем всё, возвращаем отфильтрованное.** В `tracker-data`
  всегда пишется полный snapshot (все бумаги) в **папку счёта**
  `<account_id>/YYYY-MM-DD-HH-MM.json` — это база для сравнения при следующем
  запуске. Раскладка по папкам исключает коллизии и разделяет базу для разных
  счетов; `account_id`/`account_label` дублируются и внутри файла. В ответе skill отдаёт только бумаги, где откл. от цены
  покупки > 10% И изменение относительно предыдущего snapshot > 1% (базис зависит
  от дня: другой день → по цене открытия, тот же день → по текущей цене). Пороги
  (`snapshot_deviation_threshold`, `snapshot_change_threshold`) — в `KNOWLEDGE.md`.
- **Порог алерта.** Сравнивается модуль отклонения
  `(current − averagePrice) / averagePrice`; знак задаёт направление
  (`down`/`up`). По умолчанию `threshold=0.10`, `direction=both`.
- **Разделение ответственности (Telegram).** Дифф и цифры — `tracker`
  (детерминированно). Текст сообщения — агент `telegram-message` на дешёвой модели
  (Haiku), только оформление. Отправка — skill `telegram_alert` (обёртка над
  `send.py`), без логики. Так дорогая модель не тратится на генерацию текста, а
  арифметика гарантирована (её делает не Haiku).
- **Telegram-доставка.** `send.py` → Bot API `sendMessage`; доступы из `.env`
  (`TELEGRAM_BOT_TOKEN`/`TELEGRAM_CHAT_ID`). В облаке нужен доступ к
  `api.telegram.org` (Network access = Custom).

## Проверено на реальных данных

Счёт `869168` (8 позиций). Для шага 3 при пороге 10% в обе стороны срабатывают
6 из 8 бумаг: BAZA (−16.7%), DGTL (−16.9%), SMLT (−63.7%), RENI (−11.5%) —
падение; OZON (+93.7%), LQDT (+54.9%) — рост. MOEX (−8.9%) и YDEX (−8.8%) ниже
порога и не срабатывают.
