# telegram-alert

**Только отправка.** Принимает готовый текст и отправляет его в Telegram скриптом
`send.py`. Никакой логики: не резолвит счета, не считает отклонения, не обращается
к Finam MCP. Всё это делает вызывающая сторона (напр. skill `tracker`), а сюда
приходит уже готовое сообщение.

## Вход

- `text` — готовый текст сообщения (обязательный).
- `chat_id` — опционально, переопределить чат; иначе `send.py` берёт
  `TELEGRAM_CHAT_ID` из `.env`/env (имена — в `KNOWLEDGE.md`).

## Выход

```json
{ "sent": true, "message_id": 5 }
```

- `sent` — `true`, если `send.py` завершился успехом; иначе `false` с полем
  `error`.
- `message_id` — id отправленного сообщения (из вывода `send.py`).

## Шаги

1. Если `text` пуст — ничего не отправляй, верни
   `{ "sent": false, "error": "Пустой текст" }`.
2. Отправь текст, передав его на **stdin** скрипта `send.py` (в корне
   репозитория). Токен/чат `send.py` берёт из `.env`/env сам:
   ```bash
   python3 send.py <<'MSG'
   <text>
   MSG
   ```
   Если задан `chat_id` — добавь флаг `--chat-id <chat_id>`.
3. По результату: код `0` и строка `OK: сообщение <id> …` →
   `{ "sent": true, "message_id": <id> }`; иначе
   `{ "sent": false, "error": "<stderr>" }`.

## Замечания

- Секреты (токен) не логируй — `send.py` их не печатает.
- В облачной сессии нужен доступ к `api.telegram.org` (см. `KNOWLEDGE.md`,
  раздел Telegram).
