# alert

Финальный шаг пайплайна отслеживания цен портфеля. По одному счёту Финам находит
бумаги, у которых **текущая цена отклонилась от средней цены покупки
(`averagePrice`) на порог и более**, и отправляет сводку сработавших бумаг в
**Telegram**.

Для текущей цены по каждой бумаге **вызывает skill `extract_price`** — не
дублирует его логику и не обращается к `get-quote` напрямую. Среднюю цену покупки
берёт из `get-account` (единственный источник цены покупки).

## Настройки

Значения по умолчанию бери из `KNOWLEDGE.md` в корне репозитория (порог
`threshold`, направление `direction`, счёт по умолчанию, env-имена Telegram-
доступов). Явно переданные аргументы имеют приоритет над `KNOWLEDGE.md`.

## Вход

- `account` — идентификатор счёта: **ID** (напр. `869168`), **название** (`label`)
  или торговый код (`tradeCode`). Так же, как в skill `tracker`. Если не передан —
  счёт по умолчанию из `KNOWLEDGE.md`.
- `threshold` — порог срабатывания в долях. По умолчанию — из `KNOWLEDGE.md`
  (`0.10`, т.е. 10%).
- `direction` — какое отклонение считать срабатыванием. По умолчанию — из
  `KNOWLEDGE.md` (`both`):
  - `both` — любое отклонение ≥ порога (и рост, и падение);
  - `down` — только падение (current ниже averagePrice на ≥ порога);
  - `up` — только рост (current выше averagePrice на ≥ порога).
- **Telegram-доступы** (для отправки сообщения) — из env, имена переменных
  заданы в `KNOWLEDGE.md`:
  - `bot_token` — токен бота, иначе из env `TELEGRAM_BOT_TOKEN`;
  - `chat_id` — идентификатор чата/канала, иначе из env `TELEGRAM_CHAT_ID`.

## Выход

1. **JSON-результат:**

```json
{
  "account_id": "869168",
  "generated_at": "2026-07-14T12:30:00Z",
  "threshold": 0.10,
  "direction": "both",
  "alerts": [
    {
      "ticker": "BAZA@MISX",
      "average_price": 113.29,
      "current": 94.42,
      "change_pct": -16.66,
      "direction": "down"
    }
  ],
  "telegram": { "sent": true, "chat_id": "123456789" }
}
```

- `alerts` — только сработавшие бумаги (|change_pct| ≥ threshold·100 с учётом
  `direction`). Пустой массив — значит ни одна бумага не превысила порог.
- `change_pct` — отклонение в процентах: `(current − average_price) / average_price · 100`,
  знак сохраняется (отрицательный — падение, положительный — рост). Округли до
  2 знаков.
- `direction` элемента — `down` при отрицательном отклонении, `up` при
  положительном.
- `telegram.sent` — `true`, если сообщение отправлено; `false` с полем `error`,
  если доступы не заданы или отправка не удалась.

2. **Markdown-таблица** сработавших бумаг (для читаемости):

| Тикер     | Покупка | Текущая | Δ %    |
|-----------|--------:|--------:|-------:|
| BAZA@MISX |  113.29 |   94.42 | −16.66 |

## Шаги

1. **Резолв счёта.** Как в skill `tracker`: вызови `get-accounts-list`
   (`mcp__Finam_MCP__get-accounts-list`) и сопоставь вход `account`:
   - точное совпадение с `id`/`account_id` элемента → его `id`;
   - иначе регистронезависимое совпадение с непустым `label` → его `id`;
   - иначе совпадение с `tradeCode` → его `id`;
   - если ничего не совпало — верни `{ "error": "Счёт не найден", "account": <вход> }`.
2. **Позиции с ценой покупки.** Вызови `get-account`
   (`mcp__Finam_MCP__get-account`) с найденным `account_id`.
   - Если ответ содержит `error` — верни `{ "account_id": ..., "error": "<сообщение>" }`.
   - Если `positions` отсутствует/пуст — верни результат с пустым `alerts`
     (сообщение в Telegram не отправляй).
   - Иначе по каждой позиции возьми `symbol` и `averagePrice`. В ответе Finam MCP
     это **вложенные объекты**: `averagePrice.value` — строка (напр. `"113.29"`).
     Приведи к числу. Позиции с отсутствующим/нулевым `averagePrice` пропусти
     (делить на ноль нельзя).
3. **Текущая цена через skill `extract_price`.** Для КАЖДОГО `symbol`:
   - вызови skill `extract_price` через инструмент **Skill**
     (`skill: extract_price`, `args: <symbol>`);
   - возьми из результата `current`. Если `extract_price` вернул `error` или
     `current` пуст — пропусти бумагу (в алерты не добавляй, не прерывай обход).
   Не вызывай `get-quote` и не повторяй логику `extract_price`.
   > Примечание: у `get-account` тоже есть `currentPrice`, но источником текущей
   > цены в пайплайне является `extract_price` — используем именно его.
4. **Считай отклонение.** Для каждой бумаги:
   - `change = (current − average_price) / average_price`;
   - `change_pct = round(change · 100, 2)`;
   - сработало, если `abs(change) ≥ threshold` И направление подходит под
     `direction` (`both` — всегда; `down` — только `change < 0`; `up` — только
     `change > 0`).
   Сработавшие добавь в `alerts` с полями из раздела «Выход».
5. **Отправь в Telegram.** Если `alerts` не пуст:
   - собери текст сообщения, напр.:
     ```
     ⚠️ Alert по счёту 869168 (порог 10%)
     🔴 BAZA@MISX: 113.29 → 94.42 (−16.66%)
     🔴 SMLT@MISX: 781.2 → 283.8 (−63.67%)
     🟢 OZON@MISX: 1607.6 → 3114.5 (+93.74%)
     ```
     (🔴 — падение, 🟢 — рост);
   - определи доступы: `bot_token` (или env `TELEGRAM_BOT_TOKEN`) и `chat_id`
     (или env `TELEGRAM_CHAT_ID`). Если хотя бы один не задан — не отправляй,
     верни `telegram: { "sent": false, "error": "Не заданы bot_token/chat_id" }`,
     но JSON-результат и таблицу всё равно верни;
   - отправь через **Telegram Bot API** `sendMessage` (Bash `curl`):
     ```bash
     curl -sS -X POST \
       "https://api.telegram.org/bot${TELEGRAM_BOT_TOKEN}/sendMessage" \
       --data-urlencode "chat_id=${TELEGRAM_CHAT_ID}" \
       --data-urlencode "text=<текст сообщения>"
     ```
   - по ответу проставь `telegram.sent` (`true`, если `"ok": true`) либо
     `telegram: { "sent": false, "error": "<описание из ответа>" }`.
   - Если `alerts` пуст — ничего не отправляй, верни `telegram: { "sent": false,
     "error": "Нет сработавших бумаг" }`.
6. **Отдай результат.** Верни JSON-результат и Markdown-таблицу сработавших
   бумаг, а также статус отправки в Telegram.

## Замечания

- Порог сравнивается по модулю отклонения; знак определяет направление
  (`down`/`up`) и эмодзи в сообщении.
- Skill не пишет snapshot в `tracker-data` — за фиксацию истории отвечает skill
  `tracker`. Здесь — только сравнение с ценой покупки и уведомление.
- Один вызов = один счёт. Для нескольких счетов вызывай `alert` по каждому.
- Секреты (`bot_token`) не логируй и не включай в возвращаемый результат.
