# tracker

Собирает snapshot текущих цен по всем бумагам одного счёта Финам. Для получения
цены по каждой бумаге **вызывает skill `extract_price`** — не дублирует его логику
и не обращается к `get-quote` напрямую.

## Вход

- `account` — идентификатор счёта одним из двух способов:
  - **ID счёта** (напр. `869168`), либо
  - **название счёта** (`label`) или торговый код (`tradeCode`).

## Выход

1. **JSON-snapshot:**

```json
{
  "account_id": "869168",
  "account_label": "",
  "generated_at": "2026-07-14T12:30:00Z",
  "rows": [
    { "ticker": "BAZA@MISX", "open": 94.8, "current": 94.5, "datetime": "2026-07-14T12:28:58Z" }
  ]
}
```

- `rows` — по одной записи на бумагу, поля берутся из ответа `extract_price`.
- Если по бумаге `extract_price` вернул ошибку, строка сохраняется в виде
  `{ "ticker": "...", "error": "<текст>" }` — обход остальных не прерывается.

2. **Markdown-таблица** (для читаемости), например:

| Тикер     | Открытие | Текущая | Время                     |
|-----------|---------:|--------:|---------------------------|
| BAZA@MISX |     94.8 |    94.5 | 2026-07-14T12:28:58Z      |

## Шаги

1. **Резолв счёта.** Вызови Finam MCP `get-accounts-list`
   (`mcp__Finam_MCP__get-accounts-list`). Сопоставь вход `account`:
   - точное совпадение с `id` элемента → берём этот `id`;
   - иначе точное (регистронезависимо) совпадение с непустым `label` → его `id`;
   - иначе совпадение с `tradeCode` → его `id`;
   - если ничего не совпало — верни `{ "error": "Счёт не найден", "account": <вход> }`.
2. **Позиции.** Вызови `get-account` (`mcp__Finam_MCP__get-account`) с найденным
   `account_id`.
   - Если ответ содержит `error` (напр. «Unknown account type») — верни
     `{ "account_id": ..., "error": "<сообщение>" }`.
   - Если `positions` отсутствует/пуст — верни snapshot с пустым `rows`.
   - Иначе собери список `symbol` из `positions[]` (формат уже `TICKER@MIC`).
3. **Цены через skill `extract_price`.** Для КАЖДОГО `symbol`:
   - вызови skill `extract_price` через инструмент **Skill**
     (`skill: extract_price`, `args: <symbol>`);
   - возьми из его результата `{ ticker, open, current, datetime }` и добавь в
     `rows`. При ошибке — добавь `{ ticker, error }` и продолжай.
   Не вызывай `get-quote` и не повторяй нормализацию/парсинг из `extract_price` —
   вся логика получения цены живёт в `extract_price`.
4. **Собери snapshot.** Заполни `account_id`, `account_label` (из шага 1),
   `generated_at` (текущее время UTC ISO 8601) и `rows`.
5. **Запиши snapshot в репозиторий.** По факту формирования snapshot сохрани его
   в репозиторий `mikaelo/tracker-data`:
   - имя файла — `YYYY-MM-DD-HH-MM.json` в **корне** репозитория, где дата/время
     берутся из `generated_at` в **UTC** (напр. `2026-07-14-12-41.json`);
   - содержимое — JSON-snapshot из шага 4 (тот же объект);
   - запись через GitHub MCP `create_or_update_file` с параметрами
     `owner: mikaelo`, `repo: tracker-data`, `branch:` <ветка по умолчанию репо>,
     `path: <YYYY-MM-DD-HH-MM>.json`, `content: <JSON snapshot>`,
     `message: "snapshot <account_id> <generated_at>"`. Имя уникально в пределах
     минуты → это всегда создание нового файла, `sha` не нужен.
   - Предусловие: репозиторий `mikaelo/tracker-data` должен быть в scope сессии
     (при необходимости подключить его через `add_repo`) и доступен на запись.
     Если запись вернула ошибку доступа (403/недоступен) — сообщи об этом, но
     snapshot всё равно верни в ответе (см. ниже).
6. **Отдай результат.** Верни JSON-snapshot и Markdown-таблицу, а также статус
   записи в репозиторий (путь к созданному файлу либо текст ошибки).

## Замечания

- Snapshot фиксирует «бумага — цена открытия — текущая цена — время». Цена
  покупки (`averagePrice`) в этот snapshot не входит — сравнение с ценой покупки
  и порогом 10% выполняется отдельным шагом пайплайна.
- Один вызов = один счёт. Для нескольких счетов вызывай `tracker` по каждому.
