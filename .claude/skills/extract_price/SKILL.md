# extract_price

Получает котировку **одной** акции из Finam MCP и возвращает цену открытия,
текущую цену и время получения котировки. Это шаг «получение данных» пайплайна
отслеживания цен портфеля.

## Настройки

Значения по умолчанию бери из `KNOWLEDGE.md` в корне репозитория (биржа по
умолчанию, формат времени). Явно переданные аргументы имеют приоритет.

## Вход

- `ticker` — инструмент в формате `TICKER@MIC` (напр. `SBER@MISX`, `OZON@MISX`,
  `MOEX@MISX`).
  - Если биржа (`@MIC`) не указана, добавляется биржа по умолчанию из
    `KNOWLEDGE.md` (`@MISX`): `SBER` → `SBER@MISX`.

## Выход

JSON-объект строго следующего вида:

```json
{
  "ticker": "SBER@MISX",
  "open": 321.30,
  "current": 321.97,
  "datetime": "2026-06-12T13:30:05+03:00"
}
```

- `ticker` — нормализованный тикер `TICKER@MIC`.
- `open` — цена открытия (поле `open` из ответа `get-quote`).
- `current` — текущая цена, цена последней сделки (поле `last`).
- `datetime` — дата и время котировки **по московскому времени (МСК, UTC+3)** в
  ISO 8601 со смещением `+03:00`. Источник — `as_of` элемента `items[0].as_of`
  (если его нет — `meta.as_of`); это время в UTC, переведи его в МСК (+3 часа).

При ошибке верни объект с полем `error` и понятным пояснением, например:

```json
{ "ticker": "ZZZZ@MISX", "error": "Инструмент не найден" }
```

## Шаги

1. **Нормализуй `ticker`.** Убери пробелы. Если в строке нет символа `@`,
   добавь биржу по умолчанию из `KNOWLEDGE.md` (`@MISX`). Приведи тикер-часть к
   верхнему регистру.
2. **Вызови Finam MCP** `get-quote` (`mcp__Finam_MCP__get-quote`) с
   `symbols: [<нормализованный ticker>]`.
3. **Проверь ответ:**
   - Если `items` пуст — верни `{ "ticker": ..., "error": "Инструмент не найден" }`.
   - Если у `items[0]` есть поле `status` (объект с `code`/`message`), значит
     инструмент не отдал котировку (напр. `code: "SECURITY_NOT_FOUND"`) — верни
     `{ "ticker": ..., "error": <status.message> }`.
4. **Извлеки поля** из `items[0]`: `open` → `open`, `last` → `current`.
   Время возьми из `items[0].as_of` (или `meta.as_of`, если поле отсутствует) —
   это UTC. **Переведи его в московское время (UTC+3)** и отдай в ISO 8601 со
   смещением `+03:00` → `datetime` (напр. `2026-06-12T10:30:05Z` →
   `2026-06-12T13:30:05+03:00`). Секунды сохрани, дробную часть можно отбросить.
5. **Верни** результат в формате раздела «Выход». Не добавляй лишних полей и не
   округляй цены — отдавай значения как есть.

## Замечания

- Данные могут быть реальными или отложенными (`meta.is_delayed`). Для этого
  skill это не важно — время котировки всегда берётся из `as_of` (в UTC) и
  переводится в МСК.
- Skill работает по одному тикеру. Для набора инструментов вызывай его по
  каждому тикеру отдельно (оркестрация — на уровне вызывающего шага).
